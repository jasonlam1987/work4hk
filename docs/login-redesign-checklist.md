# 登錄頁改版測試清單

## 1. 基本互動
- 進入 `/login` 可看到新版雙欄佈局（桌面）與卡片式設計。
- `Google`、`Facebook`、`Apple` 三個按鈕點擊後會進入 loading，並顯示「尚未接入 OAuth」錯誤提示。
- 「註冊（即將開放）」按鈕顯示灰色不可點擊狀態。

## 2. 表單驗證
- 帳號為空時提交，顯示「請輸入帳號」。
- 輸入 `abc@` 後失焦，顯示「Email 格式不正確」。
- 輸入 `ab` 後失焦，顯示「使用者名稱需為 3-32 字元」。
- 密碼為空時提交，顯示「請輸入密碼」。
- 密碼少於 6 碼時提交，顯示「密碼長度至少 6 碼」。

## 3. 帳號密碼登入流程
- 輸入合法帳密後可正常呼叫 `/auth/login` 並導向 `/dashboard`。
- 若 API 回傳帳號不存在，顯示「沒有該賬號，請先註冊」。
- 若 API 回傳密碼錯誤，顯示「密碼錯誤，請重新輸入」。

## 4. Disabled 狀態
- 帳密登入送出中時，`Sign In` 與三個第三方登入按鈕都不可重複點擊。
- 任一第三方登入 loading 中時，其他第三方按鈕與 `Sign In` 不可點擊。

## 5. RWD 斷點
- `>=1024px`：左側藍色品牌區 + 右側登入卡雙欄。
- `640px ~ 1023px`：改為上下堆疊，按鈕與欄位間距保持一致。
- `<640px`：內容不溢出，輸入框、按鈕可完整操作，文字不重疊。
